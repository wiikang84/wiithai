## 🎭 역할 4단계 (현재 3단계 → 4단계로 확장)

| 역할 | 누구 | 핵심 권한 | 현재 상태 |
|---|---|---|---|
| **master** | 형님 | 전체 관리, 슈퍼바이저 임명, 모든 가게·사용자·신청 관리 | ✅ 있음(이메일 화이트리스트) |
| **supervisor** ★신규 | 영업 담당 | **현장 가게 등록**, 신규 사장 유치, 본인이 등록한 가게 관리 | ❌ 신설 필요 |
| **owner**(가게사장) | 점주 | 본인 가게 관리(영업시간·긴급휴무·메뉴·사진). **가게 2개 이상 가능** | ⚠️ 있으나 1가게 가정 → 다중 소유 확장 |
| **user**(일반) | 외국인 사용자 | 발견·저장·쿠폰 | ✅ 있음 |

## 📱 1. 현장 가게 등록 폼 (master/supervisor용) ★핵심
> "현장에서 사진 찍어서 바로 추가" — 지금의 코드/JSON 등록을 대체하는 모바일 UI.

- [ ] **모바일 카메라 촬영 업로드**: `<input capture="environment">` → 가게 외관/내부/메뉴 사진 Firebase Storage 업로드 (지금은 사진 URL 수동입력만)
- [ ] **사업자등록증 촬영/업로드**: 등록 시 첨부 (지금은 owner '신청'용만 있음 → master/supervisor 등록 폼에도)
- [ ] **기본정보 입력**: 상호·주소·카테고리·국적·전화·영업시간·주메뉴 (단건 폼 이미 있음 → 모바일 최적화 + 사진/서류 결합)
- [ ] **GPS 자동 좌표**: `navigator.geolocation`으로 현재 위치 → lat/lng 자동 (수집 데이터 좌표 공백 문제도 현장에서 해결)
- [ ] **다국어**: 한국어로 입력 → 7개 언어는 등록 후 일괄 번역(기존 파이프라인 재사용)
- [ ] 등록 즉시 stores 반영 + verified 처리(현장 확인이므로 신뢰 높음)

## 🔗 2. 가게 ↔ 사장 연동 (소유권 부여)
- [ ] master/supervisor가 등록한 가게에 **사장 초대**(이메일 또는 초대코드/QR)
- [ ] 사장이 수락하면 ownerUid 연결 → **긴급휴무·영업시간·사진** 직접 관리 (owner.html 이미 있음 — 재사용)
- [ ] **가게 2개 이상 소유**: 현재 owner는 1가게 가정 → owner가 소유한 가게 목록(stores where ownerUid==me) + 가게 선택 후 관리하도록 확장
- [ ] 긴급휴무 → 손님 발견화면 "오늘 휴무" 배지 (이미 연동됨)

## 🛡 3. 역할/권한 인프라
- [ ] **supervisor 역할 신설**: 데이터 모델(users.role 또는 별도 supervisors 컬렉션) + 화면 분기
- [ ] **Custom Claims 전환**: 지금 master는 이메일 화이트리스트(클라 노출). 정식은 Cloud Function으로 role을 토큰에 박아 rules에서 검증 → 보안·확장
- [ ] firestore/storage.rules 역할별 재작성 (supervisor 등록 권한, owner 다중 소유 검증)
- [ ] master 화면에서 supervisor 임명/해제 UI

## 🗂 4. 관리 화면 분리 (역할별)
- [ ] **master 콘솔**: 전체 가게·사용자·슈퍼바이저·신청 관리 (현 store-admin 확장)
- [ ] **supervisor 앱/화면**: 현장 등록 폼 + 본인 등록 가게 목록 + 사장 유치 현황 (모바일 우선)
- [ ] **owner 화면**(owner.html): 다중 가게 선택 + 운영 (확장)

## 📌 우선순위 제안
1. **현장 등록 폼(§1)** — 사진 촬영·GPS·서류 업로드. 이게 "코드로 안 넣고 현장에서"의 핵심이자 영업 무기. (master부터, supervisor는 §3 후)
2. **owner 다중 가게(§2)** — 사장이 여러 가게 관리. 비교적 작음.
3. **supervisor 역할 + Custom Claims(§3)** — 인프라. 영업 인원 늘릴 때 필요.
4. **관리 화면 분리(§4)** — 위가 되면 자연스럽게.

## ⚠️ 검토 필요 포인트
- **사진 저장 비용**: Firebase Storage 무료 한도(5GB). 가게당 사진 여러 장 × 수백 가게면 압축/리사이즈 필요(업로드 전 클라 리사이즈).
- **현장 등록 권한 남용 방지**: supervisor가 부실 등록 못 하게 master 검수 단계 or 사후 verified 토글.
- **Custom Claims 전환 시점**: 지금 화이트리스트로도 동작하나, supervisor가 여러 명 되면 Cloud Function 필수.

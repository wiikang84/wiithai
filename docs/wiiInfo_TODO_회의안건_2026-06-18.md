## 🗳 3팀 우선순위 회의 결론 (전략PM·마케팅·개발 종합)

**공통 메시지: "디자인 추가 다듬기는 멈춰라. 지금 병목은 픽셀이 아니라 알맹이다."**
현재 완성도 = 껍데기 거의 완성 / 알맹이 부족. 아래 순서 권고:

| 순위 | 작업 | 담당 | 근거 |
|---|---|---|---|
| **1** | **다크모드 하드코딩 전수 치환** (6개 화면) | 개발(코드) | OS 자동 다크모드라 회피 불가. 더 만들기 전에 부채 정리. ↓아래 목록 |
| **2** | **메타·OG·manifest·privacy 피벗 교체** | 개발(코드, 반나절) | 공유·검색·앱설치가 아직 "한국어 학습앱"으로 뜸. privacy에 위치정보 누락=심사 리스크 |
| **3** | **실검증 점포 데이터 10~20개** | ⚠️ 형님 현장 | 지금 places.js 5개 전부 미검증, 2개는 가짜(Sample/Feedback). "열면 진짜 가게 0개" = 발견앱 효용 0. **외주로 못 채움(해자)** |
| **4** | **점주(사장님) 기능** | 회의 중 ↓ | 형님 신규 안건. 아래 별도 섹션 |
| **5** | **획득 루프(점포 QR/공유 링크 + utm)** | 개발 | 영업 전략 핵심 도구. "어느 가게 QR이 먹히나" 유일한 측정 지표 |
| **6** | 가게 사진 + 상세 모달 재디자인 | 형님+개발 | 영업 방문 시 외관 1컷씩. 상세모달은 전환 직결 |
| **7** | 카카오맵 실동작 확인 | 형님+개발 | ↓아래 |

**마케팅 핵심**: 영업 나가기 전 최소 세트 = **메타 교체 → 그날 돌 동네 검증데이터 → QR**. 셋 중 하나라도 빠지면 첫인상 태움.
